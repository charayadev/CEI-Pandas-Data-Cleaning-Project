## 📌 Project Overview

This project was completed as part of the **Celebal Excellence Internship (CEI)** and demonstrates applied SQL skills on a structured e-commerce database. The analysis covers the full spectrum of SQL usage — from basic data exploration to advanced concepts like transactions, indexing, and ACID compliance — all performed using **PostgreSQL** and **pgAdmin 4**.

---

## 🎯 Objective

To analyse an e-commerce sales database using structured SQL queries, covering data retrieval, transformation, aggregation, multi-table joins, constraints, and transaction management — building a strong foundation in real-world database analysis.

---

## 🗄️ Database Overview

**Database:** `ecommerce_sales_db`

| Table | Description |
|---|---|
| `customers` | Customer profile and contact information |
| `orders` | Order-level data including dates and status |
| `order_items` | Line-item details linking orders to products |
| `products` | Product catalogue with pricing and categories |

## 📚 SQL Concepts Covered

- **Data Exploration** — Inspecting schema, data types, and row samples
- **Filtering** — `WHERE`, `BETWEEN`, `IN`, `LIKE`, `IS NULL`
- **Aggregation** — `COUNT`, `SUM`, `AVG`, `MIN`, `MAX`
- **Grouping & Sorting** — `GROUP BY`, `HAVING`, `ORDER BY`
- **Joins** — `INNER JOIN`, `LEFT JOIN`, multi-table joins
- **Conditional Logic** — `CASE` statements
- **Database Design** — Constraints and indexing
- **Transactions** — `BEGIN`, `COMMIT`, `ROLLBACK`
- **ACID Properties** — Atomicity, Consistency, Isolation, Durability
- **Data Validation** — Enforcing integrity at query and schema level
